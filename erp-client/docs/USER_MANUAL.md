# NEKO_ERP 客户端使用手册

更新时间: 2026-04-23
适用范围: `erp-client` Electron 桌面客户端
目标读者: 试点业务用户、实施人员、一线支持人员

## 1. 客户端定位

NEKO_ERP 客户端是当前系统的唯一主入口。

它负责下面这些事情:

- 登录和保持当前桌面会话
- 提供模块导航、列表、详情和工作台
- 通过 Electron 提供桌面通知、文件选择、打印和多窗口
- 连接本地或远程服务端 API
- 为销售、采购、出入库、发票、账户管理提供统一操作界面

## 2. 启动前准备

首次启动前，请确认下面条件已经满足:

1. 服务端包已经启动并可访问
2. PostgreSQL、Redis、RabbitMQ、MinIO 基础设施已经就绪
3. 当前电脑具备桌面版运行环境
4. 已经拿到可用的系统账号

如果拿到的是分开发放的 0.1 预览版，请按这个顺序使用:

1. 先解压并启动服务端包
2. 再解压并启动客户端包
3. 在客户端登录页确认服务地址
4. 最后再登录系统

发布前人工试跑时，建议同时打开:

- `erp-client/docs/PREVIEW_SMOKE_CHECKLIST.md`
- 发布包内的 `docs/client-smoke-checklist.md`

如果是 0.1 预览版试跑，默认后端地址通常为:

```text
http://127.0.0.1:8080/api
```

默认管理员账号:

```text
用户名: admin
密码: admin
```

## 3. 登录与连接

### 3.1 登录步骤

1. 打开桌面客户端
2. 在登录页输入用户名和密码
3. 确认连接地址正确
4. 点击登录

### 3.2 连接设置

如果后端不是本机地址，可在设置页调整:

- 服务地址
- 代理地址
- 代理绕过规则
- 是否记住用户名

### 3.3 当前会话

当前版本的登录态只保留在当前桌面会话内。

这意味着:

- 重启客户端后通常需要重新登录
- 可以在设置页刷新当前账户信息
- 可以在设置页直接修改当前账户密码

## 4. 主界面结构

客户端主要由四个区域组成:

### 4.0 预览首页

0.1 预览版建议先看首页里的“首跑检查清单”。

它会帮助你快速确认:

- 服务端包是否在线
- 当前账户是否已经进入桌面会话
- 首批应该先试跑哪些模块
- 每一步可以直接点哪里进入
- 哪些步骤已经留下最近对象，可直接重开继续处理

当前检查清单常见状态:

- `已就绪` / `检查中` / `未连接`: 用于服务端连通性
- `待首跑`: 该模块已经开放，但还没有首条可用记录
- `已留痕`: 该模块已经有最近对象，可直接点“重开最近对象”
- `已关闭`: 该模块被关闭，需要先去设置页恢复

建议按这个顺序完成第一次试跑:

1. 确认服务端包在线
2. 确认当前会话和管理员账户
3. 新建公司主档
4. 新建伙伴主档
5. 新建产品模板
6. 新建产品变体
7. 新建销售订单
8. 新建采购订单
9. 核查账单链路
10. 检查用户和角色管理是否可交付给其他人

### 4.1 左侧导航

左侧导航用于打开模块。

当前已经支持:

- 一级分类展开和折叠
- 多分类同时展开
- 分类展开状态记忆
- 模块图标显示
- 首批主链模块快速进入

进入模块后，如果列表还没有任何数据，页面会直接显示首跑入口。

常见动作包括:

- 新建第一条记录
- CSV 批量导入
- 导出导入模板
- 重开最近对象
- 打开切换设置

### 4.2 顶部工作区

顶部区域主要显示:

- 当前页面标题
- 当前连接状态
- 当前用户
- 常用动作入口

### 4.3 列表区

列表区主要用于:

- 查询模块数据
- 按关键字过滤
- 浏览记录摘要
- 选择目标记录

### 4.4 详情工作区

详情工作区是日常操作的主战场。

当前详情页已经逐步收敛为:

- 一个对象一个主操作区域
- 详情和列表分工明确
- 重要对象支持直接进入独立详情上下文
- 附件、时间轴、关联对象集中显示

## 5. 通用操作

### 5.1 查询和过滤

每个模块通常都支持:

- 关键字查询
- 常见字段过滤
- 列表刷新
- 重置当前过滤条件

### 5.2 新建记录

进入模块后，可通过下面方式新建:

- 页面内点击新建
- 使用快捷键 `Ctrl+N`
- 部分场景通过命令台直接快速新建

### 5.3 保存记录

保存当前表单时，可使用:

- 页面保存按钮
- 快捷键 `Ctrl+S`

进入详情页后，标题下方会出现“下一步动作”提示条。

它会根据当前记录状态提示你:

- 直接执行下一步业务动作
- 打开文档区
- 导入业务文件
- 打开流程区
- 导出当前工作区

### 5.4 返回列表

当你位于详情页或新建页时，可用下面方式回到列表:

- 页面内返回动作
- 快捷键 `Alt+Left`
- 也可以先回首页，再通过“首跑检查清单”或“试点启动台”重开最近对象

### 5.5 打印和导出

客户端已经集成桌面打印与导出能力。

常用方式:

- `Ctrl+P`: 打印当前工作区
- `Ctrl+Shift+E`: 导出当前工作区
- `Ctrl+Alt+E`: 导出当前列表

### 5.6 CSV 导入

部分模块支持 CSV 导入流程。

使用方式:

- 进入模块
- 调用导入动作或快捷键 `Ctrl+Alt+I`
- 选择 CSV 文件
- 按当前模块语义导入

### 5.7 附件、合同、发票

详情页支持把业务资料挂到对象上。

常见资料包括:

- 合同
- 发票扫描件
- 对账附件
- 交付证明
- 客户往来文件

文件导入通常通过桌面文件选择器完成，适合直接从本机导入资料。

### 5.8 Timeline 时间轴

部分核心对象支持 Timeline。

时间轴用于聚合:

- 系统日志
- 人工便签
- 附件记录

建议在销售、采购和伙伴对象上持续维护上下文，避免信息只留在聊天工具里。

## 7. 命令台与快捷键

### 7.1 命令台

命令台用于快速搜索模块、记录和动作。

默认快捷键:

```text
Ctrl+K
```

适合下面场景:

- 快速打开模块
- 快速跳转最近对象
- 快速执行关键动作

### 7.2 默认快捷键

- `Ctrl+K`: 打开命令台
- `Ctrl+,`: 打开设置
- `Alt+Left`: 返回列表
- `Ctrl+N`: 新建记录
- `Ctrl+S`: 保存当前表单
- `Ctrl+Shift+E`: 导出当前工作区
- `Ctrl+P`: 打印当前工作区
- `Ctrl+Alt+E`: 导出当前列表
- `Ctrl+Alt+I`: 导入 CSV
- `F5`: 刷新当前页面

### 7.3 自定义快捷键

快捷键可以在设置页中调整。

当前版本不是手动输入文本，而是:

1. 点击对应快捷键输入框
2. 直接按下新的键盘组合
3. 系统自动捕获并保存

说明:

- 使用 `Ctrl` 的组合在 macOS 上会自动兼容 Command
- 改动通常立即生效

## 8. 设置页说明

设置页当前重点包含下面几个部分:

### 8.1 常规与外观

- 语言
- 主题
- 字体缩放
- 界面密度
- 自定义颜色

### 8.2 连接与会话

连接页顶部现在提供 `0.1 基础管理启动台`。

它的用途不是展示技术信息，而是把预览版最常用的基础管理入口收在一起:

- 当前后端目标
- 当前桌面账户
- 用户管理入口
- 角色管理入口
- 快捷键设置入口
- 硬件加速与重启要求提示

建议第一次交付试点人员时，先从这里确认:

1. 服务端地址是否正确
2. 当前管理员账号是否可用
3. 用户和角色主档是否已经建立
4. 常用快捷键是否适合当前团队
5. 是否存在“已保存但需要重启”的桌面运行设置

- 后端地址
- 代理
- 当前登录用户
- 刷新当前会话
- 退出当前会话
- 修改当前密码

如果需要做账户和权限交接，建议按下面顺序操作:

1. 先在连接页刷新当前账户
2. 再打开用户管理维护账号
3. 然后打开角色管理维护角色编码和启停状态
4. 最后回到快捷键页，为操作员录制高频动作组合键

### 8.3 通知

- 桌面通知
- 提示音
- 每日摘要

### 8.4 快捷键

- 查看当前绑定
- 录制新快捷键
- 单项恢复默认
- 全部恢复默认

### 8.5 硬件加速

客户端支持硬件加速开关。

使用建议:

- 普通办公电脑建议开启
- 如果出现黑屏、卡顿、闪烁，可尝试关闭

重要说明:

```text
硬件加速相关设置保存后，需要重启软件才会真正生效。
```

## 9. 常见工作流建议

### 9.1 伙伴维护

建议维护下面信息:

- 基本身份信息
- 联系方式
- 所属公司
- 关系背景备注
- 合同和往来附件

### 9.2 销售订单处理

推荐顺序:

1. 创建或确认客户资料
2. 新建销售订单
3. 补全订单行和关键金额
4. 关联附件和合同
5. 进入后续出库和开票链路

### 9.3 采购订单处理

推荐顺序:

1. 确认供应商资料
2. 新建采购订单
3. 补全采购行
4. 附加报价单或合同
5. 继续进入收货和账单链路

## 10. 常见问题

### 10.1 登录失败

优先检查:

- 用户名和密码是否正确
- 用户是否被停用
- 服务端包是否已经启动
- 当前连接地址是否正确
- 地址是否以 `/api` 结尾

### 10.2 页面打不开或数据为空

优先检查:

- 后端 API 是否在线
- 当前服务端包和中间件是否已经全部启动
- 当前模块是否被切换开关限制
- 当前过滤条件是否过严
- 当前账号是否已失效

### 10.3 文件无法导入

优先检查:

- 文件是否被其他程序占用
- 文件格式是否符合当前模块要求
- 本机是否有读取权限

### 10.4 修改了 GPU 选项但没变化

这是正常现象。

硬件加速设置需要重启客户端后才会真正应用。

## 11. 当前版本边界

0.1 预览版的定位是试点运行，不是最终生产版。

当前使用建议:

- 先用小范围团队试跑
- 每条业务链明确负责人
- 保留回退路径
- 重要资料务必挂附件并保留追溯

不建议当前阶段直接把它当作全量正式生产替代方案。
