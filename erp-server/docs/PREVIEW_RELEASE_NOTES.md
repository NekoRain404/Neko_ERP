# NEKO_ERP 服务端 0.1 预览版发布说明

更新时间: 2026-04-23
适用范围: `erp-server` Spring Boot 服务端
版本: `0.1.0-preview.1`

## 1. 本次发布的定位

本次发布是 `0.1 预览版` 服务端，不是正式生产发布。

目标是:

1. 为桌面客户端提供稳定可连接的 API
2. 支撑首批主线试跑:
   - 主数据
   - 销售
   - 采购
   - 调拨
   - 账单
   - 基础账户管理
3. 提供可配置、可排错、可回退的试点后端包

## 2. 本次发布已包含的交付内容

- `erp-server.jar`
- `settings.conf`
- 启动脚本:
  - `start-server.sh`
  - `start-server.bat`
- 服务端使用手册
- 服务端部署清单
- 预览版 docker compose 示例

## 3. 运行前提

服务端包仍然依赖外部基础设施:

- PostgreSQL
- Redis
- RabbitMQ
- MinIO

默认接口根地址:

```text
http://127.0.0.1:8080/api
```

## 4. 发布前最少核查项

1. `settings.conf` 中的端口和数据库地址是否正确
2. PostgreSQL / Redis / RabbitMQ / MinIO 是否已启动
3. `erp-server.jar` 是否可成功启动
4. Flyway 迁移是否可正常执行
5. 登录接口是否可用
6. 核心列表接口是否可访问
7. 客户端是否可成功连接当前服务端

## 5. 已知限制

- 当前不是高可用部署方案
- 当前没有正式生产级运维编排与监控方案
- 当前目标是支撑试点和演示，不是全量正式投产
- 深会计、深制造、深报表能力不属于本次发布范围

## 6. 试点支持建议

- 服务端与客户端分开发放
- 先发服务端包，再发客户端包
- 先让试点团队在单一环境试跑，不要同时多环境扩散
- 保留 `settings.conf` 原始副本，避免试点现场改乱配置后无法回退
