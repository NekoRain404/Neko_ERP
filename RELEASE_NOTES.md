# NEKO_ERP 0.1.0-preview.1 最终发布说明

更新时间: 2026-04-23
版本标签: `v0.1.0-preview.1`
发布定位: 小范围试点预览版，不是正式生产版

## 1. 本次发布的目标

本次 `0.1.0-preview.1` 不是“全量 ERP 正式上线”，而是为了尽快交付一个可安装、可登录、可试跑、可排错的桌面 ERP 预览版本。

本次重点是:

1. 桌面客户端与服务端分包交付
2. 主数据、销售、采购、库存、账单、基础账户管理可闭环试跑
3. 发布包、部署清单、试跑清单、门禁报告齐备
4. 保持可回退、可定位、可继续迭代

## 2. 本次发布包含的交付物

推荐直接分发总交付包:

- `dist/NEKO_ERP-preview-delivery-v0.1.0-preview.1.tar.gz`

组件级产物:

- `dist/NEKO_ERP-client-v0.1.0-preview.1.tar.gz`
- `dist/NEKO_ERP-server-v0.1.0-preview.1.tar.gz`

1. 客户端包
2. 服务端包
3. 最终门禁报告
4. 客户端试跑清单
5. 服务端部署清单

## 4. 推荐试点发包顺序

1. 先把总交付包发给试点负责人
2. 由试点负责人先部署服务端包
3. 服务端连通确认后，再向操作员发客户端包
4. 让试点团队按清单完成首跑

## 5. 当前范围内推荐验证的业务路径

1. 登录
2. 首页首跑检查清单
3. 公司主档
4. 伙伴主档
5. 产品模板
6. 产品变体
7. 销售订单
8. 采购订单
9. 账单查看
10. 用户与角色管理
11. 详情页附件与时间轴
12. 设置页快捷键与硬件加速提示

## 6. 当前明确不承诺的范围

下面这些不属于本次预览版的交付承诺:

1. 深会计
2. 深制造
3. 深 BI / 报表体系
4. 正式生产级 HA / 监控 / 编排
5. 全公司大范围铺开

## 7. 发布前最后的人工动作

虽然工程门禁已经通过，但正式把预览版发给试点团队前，仍建议至少做一次人工确认:

1. 在干净环境按交付包清单启动
2. 使用 `admin / admin` 完成一次桌面首跑
3. 确认登录、列表、详情、附件、设置、退出无明显阻塞
4. 固定本次试点交付的负责人和反馈入口

## 8. 结论

`v0.1.0-preview.1` 现在已经达到“可发给小范围试点团队”的状态。

下一阶段不应再继续发散加功能，而应优先收集试点反馈，确认哪些问题是真正阻塞 `0.1.x` 继续推进的主线问题。
